# NIP-29 Reference Card

### REFERENCE_CARD
NIP-29 defines relay-based groups with a random string _id_. Groups can be public or private, with elevated access for users defined by role labels. 
- **Kind numbers**: 
  - `9021`: join request
  - `9022`: leave request
  - `9000-9020`: moderation events (e.g., `put-user`, `remove-user`, `edit-metadata`)
  - `39000`: group metadata
  - `39001`: return group admins
  - `39003`: define roles and their capabilities
- **Required/optional tags**:
  - `h`: group _id_ (required for user-created events)
  - `d`: group _id_ (required for group metadata events)
  - `p`: pubkey hex (required for `put-user`, `remove-user`)
  - `previous`: timeline references (optional, but recommended for at least 3)
- **Filter patterns**: 
  - Events with `h` tag for group messages
  - Events with `d` tag for group metadata
- **Common use cases**:
  - Creating and managing groups
  - Sending messages to groups
  - Assigning roles to users
  - Managing group metadata (e.g., name, picture, about)

Example of a `kind:9021` join request event:
```json
{
  "kind": 9021,
  "content": "optional reason",
  "tags": [
    ["h", "<group-id>"],
    ["code", "<optional-invite-code>"]
  ]
}
```
