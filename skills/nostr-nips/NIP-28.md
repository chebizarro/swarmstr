# NIP-28 Reference Card

### REFERENCE_CARD
NIP-28 defines event kinds for public chat channels, messages, and moderation. It reserves kinds 40-44 for immediate use. 
- Kind 40: Create channel with `name`, `about`, `picture`, and `relays` metadata.
- Kind 41: Update channel metadata, ignoring updates from non-kind 40 pubkeys.
- Kind 42: Send text messages to channels, using "e" tags for relay and message type.
- Kind 43: Hide messages, optionally including a reason.
- Kind 44: Mute users, optionally including a reason.
Common use cases include creating and managing public chat channels, sending and moderating messages, and implementing client-side moderation.
Example JSON for kind 40:
```json
{
  "content": "{\"name\": \"Demo Channel\", \"about\": \"A test channel.\", \"picture\": \"https://placekitten.com/200/200\", \"relays\": [\"wss://nos.lol\", \"wss://nostr.mom\"]}",
}
```
Example JSON for kind 41:
```json
{
  "content": "{\"name\": \"Updated Demo Channel\", \"about\": \"Updating a test channel.\", \"picture\": \"https://placekitten.com/201/201\", \"relays\": [\"wss://nos.lol\", \"wss://nostr.mom\"]}",
  "tags": [
    ["e", <channel_create_event_id>, <relay-url>, "root"],
    ["t", <category_name-1>],
  ],
}
```
Filter patterns include using "e" tags for relay and message type, and "t" tags for category names.
