# NIP-50 Reference Card

NIP-50 introduces a `search` filter field for `REQ` messages. The `search` field is a string describing a query, e.g. "best nostr apps". Relays interpret the query and return matching events, prioritizing `content` field matches. Results are sorted by matching score, not `created_at`. The `limit` filter is applied after sorting. Query strings can include `key:value` pairs for extensions, such as `include:spam`, `domain:<domain>`, `language:<language_code>`, `sentiment:<sentiment>`, and `nsfw:<boolean>`. Clients can specify multiple search filters and restrict results to particular event kinds using `kinds`, `ids`, and other filter fields. Example: `["REQ", "", { "search": "orange" }, { "kinds": [1, 2], "search": "purple" }]`.
