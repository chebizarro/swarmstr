# NIP-51 Reference Card

NIP-51 defines lists of things that users can create. Lists can contain public or private references. Public items are in the event `tags` array, while private items are in a JSON array in `.content`, encrypted using NIP-44. Standard lists have special meaning and use normal replaceable events (e.g., Follow list: kind 3, Mute list: kind 10000). Sets have well-defined meaning and can enhance client functionality (e.g., Relay sets: kind 10012, Curation sets). Required tags: `kind`, `tags` (for public items). Optional tags: `d` (for sets), `title`, `image`, `description`. Filter patterns: `kind:<number>`, `tags:<item>`. JSON example: `{"kind": 3, "tags": ["p:npub123...", "p:npub456..."]}`.
