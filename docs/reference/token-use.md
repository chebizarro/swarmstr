# Token Usage & Prompt Caching

## Token Tracking

metiq tracks token usage per session and per agent turn. Token counts are stored in session transcripts and surfaced via the status command.

### Viewing Usage

```bash
metiq status --usage
metiq models list
```

The status output includes:
- Tokens used in current session
- Total tokens across all sessions (optional)
- Provider usage/quota (if provider supports it)

### Per-Session Token Count

Token counts are stored in the session transcript JSONL:

```jsonl
{"role":"assistant","content":"...","tokenCount":{"input":1500,"output":320,"cacheRead":800,"cacheWrite":0}}
```

Fields:
- `input`: tokens in the prompt (including context)
- `output`: tokens generated by the model
- `cacheRead`: tokens read from cache (cost reduced)
- `cacheWrite`: tokens written to cache

## Prompt Caching

### Anthropic Prompt Caching

Anthropic's API supports caching up to 4 cache breakpoints. metiq uses this to reduce costs on repeated context (bootstrap files, workspace files).

Configure:

```json5
{
  "agents": {
    "defaults": {
      "models": {
        "anthropic/claude-opus-4-5": {
          "params": {
            "cacheRetention": "short"   // "none" | "short" (5min) | "long" (1hr)
          }
        }
      }
    }
  }
}
```

**Default**: `cacheRetention: "short"` when using an Anthropic API key.

Cache breakpoints are inserted at:
1. System prompt end
2. Bootstrap files end (AGENTS.md, SOUL.md, etc.)
3. Memory files end
4. Tool definitions end

### Cache Hit Rates

After the first turn in a session, subsequent turns typically cache:
- System prompt: ~100% cache hit (identical across turns)
- Bootstrap files: ~100% cache hit
- Session history: grows with each turn, eventual recompaction

Expect 50-80% input token cost reduction on cached content.

## Cost Estimation

| Model | Input (per 1M) | Output (per 1M) | Cache read |
|-------|---------------|-----------------|------------|
| claude-opus-4-5 | $15 | $75 | $1.50 |
| claude-sonnet-4-5 | $3 | $15 | $0.30 |
| claude-haiku-4-5 | $0.25 | $1.25 | $0.025 |
| gpt-4o | $2.50 | $10 | $1.25 |

For a typical agent session (100 turns, 2k tokens/turn avg):
- Without caching: ~200k tokens input @ $3/1M = ~$0.60
- With caching: ~50k non-cached + 150k cached @ $0.30/1M = ~$0.20

```bash
# Check provider quota status
metiq models list --check

# Exits 0 if healthy, 1 if expired/missing, 2 if expiring soon
```

## Reducing Token Usage

### Auto-Compaction

Compaction replaces session history with a summary, reducing tokens for long sessions. See [Compaction](/concepts/compaction).

### Heartbeat Suppression

The `HEARTBEAT_OK` sentinel prevents the model from generating a response for routine heartbeats (which would consume tokens for nothing). See [Heartbeat](/gateway/heartbeat).

### Cheaper Models for Automation

Use cheaper models for cron jobs and heartbeat processing:

```json5
{
  "cron": {
    "defaultModel": "anthropic/claude-haiku-4-5"
  }
}
```
