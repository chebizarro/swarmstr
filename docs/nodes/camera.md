# Camera & Images

metiq supports camera input from paired node devices and image understanding via vision-capable models.

## Camera Node

When a node with camera capabilities is paired, the agent can request snapshots for visual context.

### Taking a Snapshot

Camera commands are sent to nodes via `metiq nodes invoke`:

```bash
# Capture a photo from a paired node
metiq nodes invoke --node <node-id> --command camera.snap

# With options
metiq nodes invoke --node <node-id> --command camera.snap \
  --args '{"max_width": 1280, "quality": 0.8}'
```

### Camera List

```bash
# List cameras available on a node
metiq nodes invoke --node <node-id> --command camera.list
```

### Select Specific Camera

```bash
metiq nodes invoke --node <node-id> --command camera.snap \
  --args '{"device_id": 1}'
```

The `--node` flag takes the node ID from `metiq nodes list`.

## Image Understanding

The agent can understand images using vision-capable models (Claude, GPT-4V, Gemini).

### How It Works

1. Camera snapshot is taken (or image received via Nostr DM)
2. Image is passed to the vision model as part of the agent turn
3. Agent describes, analyzes, or acts on the visual content

### Supported Image Sources

- Camera snaps from paired nodes
- Images sent via Nostr DMs (if the client supports image attachments)
- Screenshots from the browser tool
- Images generated by agent skills

### Vision Model Configuration

Vision is used automatically when the configured model supports it. Set the model in `config.json`:

```json
{
  "agent": {
    "default_model": "anthropic/claude-opus-4-5"
  }
}
```

Vision is supported by:
- `anthropic/claude-*` (all Claude 3+ models)
- `openai/gpt-4o`
- `openai/gpt-4-vision-preview`
- `google/gemini-*`

## Raspberry Pi Camera Setup

For a Raspberry Pi camera module:

```bash
# Enable camera
sudo raspi-config
# → Interface Options → Camera → Enable

# Install libcamera tools
sudo apt install libcamera-tools

# Test
libcamera-still -o test.jpg
```

The node device runs its own metiqd instance configured to pair with your main daemon.

## USB Camera

```bash
# Check USB cameras
ls /dev/video*

# Test with v4l2
sudo apt install v4l-utils
v4l2-ctl --list-devices
```

The node host uses v4l2 for USB camera access on Linux.

## Video Clip Capture

```bash
# Capture a short video clip
metiq nodes invoke --node <node-id> --command camera.clip \
  --args '{"duration_ms": 5000}'
```

Video clips are transcoded and passed to the vision model for analysis (provider must support video).

## Example: Security Camera Skill

A skill that periodically captures a camera snapshot and analyzes it for anomalies:

```markdown
On heartbeat or when asked, take a camera snapshot from the paired node named "securitycam".
Analyze the image for:
- Unusual activity
- Motion in restricted areas
- Package delivery

If anything notable, send a Nostr DM to the configured contact with the description and image.
```
